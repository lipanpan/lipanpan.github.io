rabbitmq 配置说明

发表于 2017-02-27   |   分类于 rabbitmq

前言

本篇主要介绍rabbitmq相关的配置，首先在rabbitmq中有两类配置：rabbitmq-env.conf环境变量配置和rabbitmq.config服务节点配置，这些文件默认存放位置是在${rabbitmq安装路径}/etc/rabbitmq目录下，默认是没有的，需要自己手动创建。下面将着重介绍这两类配置，它们有哪些配置项? 以及这些配置项的作用?

rabbitmq-env.conf

原文：In order of preference, the startup scripts get their values from the environment, from /etc/rabbitmq/rabbitmq-env.conf and finally from the built-in default values.

翻译：rabbitmq服务按照优先级顺序读取环境变量值，优先读取系统环境变量，然后读取rabbitmq-env.conf配置文件，最后才是系统预定的默认值。

RabbitMQ environment variable names have the prefix RABBITMQ_. A typical variable called RABBITMQ_var_name is set as follows:

  1. a shell environment variable called RABBITMQ_var_name is used if this is defined;
  2. otherwise, a variable called var_name is used if this is set in the rabbitmq-env.conf file
  3. otherwise, a system-specified default value is used.

上述描述传达了两个重要信息：

  1. 系统环境变量配置都是以RABBITMQ_前缀开头的，但是在rabbitmq-env.conf是不需要该前缀的；
  2. 配置的优先级：系统环境变量 > rabbitmq-env.conf配置 > 系统默认值

rabbitmq.config

下面是从github下的rabbitmq服务完整的配置信息及格式并且附有详情地址，绝大多数情况下，我们不用配置的这么详细，因为rabbitmq配置都是有默认值的，可以在rabbitmq节点下运行./rabbitmqctl status命令查看当前节点的配置信息。rabbitmq.config是一个标准的erlang配置文件，配置必须符合erlang标准规范，不然会出现配置解析报错，导致服务无法启动。以下就是个完整而标准的配置，我们可以直接在上面进行修改和调整即可！
